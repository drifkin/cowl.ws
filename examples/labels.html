Labels

// Information sensitive to a.com:
var a = new Label("http://a.com");
// Information sensitive to b.com:
var b = new Label("http://b.com");
// Information sensitive to either a.com or b.com:
var aORb = new Label("http://a.com").or("http://b.com");
// Information sensitive to both a.com or b.com:
var aANDb = new Label("http://a.com").and("http://b.com");

// Trivial check:
a.subsumes(a)    === true;

// Origins a.com and b.com are incompatible, so neither subsumes the other:
a.subsumes(b)    === false;
b.subsumes(a)    === false;

// Label b.com is more restricting than a.com or b.com -- it only allows for
// information labeled as such to be read by b.com (vs. both a.com and b.com):
b.subsumes(aORb) === true;

// Of course, the converse is not true:
aORb.subsumes(a) === false;

// Label that protects a.com and b.com, on the other hand, is more restricting 
// than just a.com or b.com:
aANDb.subsumes(a)    === true;
aANDb.subsumes(b)    === true;
aANDb.subsumes(aORb) === true;

// Again, the converse is not true since it may violate the other's privacy:
a.subsumes(aANDb)    === false;
You will notice that if you call toString() on a label it will print out in terms of Roles. Indeed you can use the Role constructor to create the disjuctive (or) parts of a label if you do not wish to use strings.

Privileges

// Create fresh privileges (origins)
var priv1 = new FreshPrivilege();
var priv2 = new FreshPrivilege();

// Fresh privileges do not subsume anything but themseves:
priv1.subsumes(priv2) === false;
priv2.subsumes(priv2) === true;

// Privileges can be combined:
priv1.and(priv2);
priv1.subsumes(priv2) === true;

// You can also extract the label that this privilege corresponds to:
priv1.asLabel;
